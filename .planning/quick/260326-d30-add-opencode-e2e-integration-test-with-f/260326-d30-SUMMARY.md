# Phase quick Plan 260326-d30: OpenCode E2E Integration Test with Failure Artifact Bundle Summary

**One-liner:** OpenCode e2e test (`test_agent_rulez_opencode_capture_to_export`) with `_collect_failure_artifacts()` producing tarball+FAILURE_REPORT.md diagnostic bundle on failure.

## What Was Built

Extended `tests/integration/test_agent_rulez_e2e.py` with full OpenCode coverage for the capture-to-export pipeline, matching the existing Claude test structure.

### New Components

**`_probe_opencode()`** — Binary probe that checks `shutil.which("opencode")` then runs a minimal headless invocation (`opencode run --dir /tmp "echo hello"`) to confirm the binary can run headlessly. Returns True only if both checks pass.

**`opencode_available` fixture** — Session-scoped fixture wrapping `_probe_opencode()` result. Used as a skip gate in the e2e test.

**`opencode_integration_env` fixture** — Fully isolated environment for OpenCode testing. Installs the plugin with `--for opencode`, creates isolated HOME with auth copied, copies the fixture project, and locates `setup.sh` from the versioned cache dir. Returns same dict shape as `integration_env`.

**`run_opencode()`** — CLI helper running `opencode run --dir {project} {prompt}` with positional message args. No `--command` flag (hangs silently per Phase 13 decision). Sets HOME override and cwd to project directory.

**`_collect_failure_artifacts()`** — Comprehensive failure bundle that:
- Creates timestamped dir under `artifacts/`
- Copies full project snapshot via `shutil.copytree`
- Captures version info (rulez, opencode, pip show, python, platform.platform())
- Writes subprocess stdout/stderr/returncode for all phases
- Copies config files (.claude/hooks.yaml, settings.json, .opencode/, capture-session.sh)
- Runs diagnostics (`rulez validate`, `rulez opencode doctor`)
- Generates recursive dir tree via `os.walk`
- Copies session JSON and docs
- Writes `FAILURE_REPORT.md` with failure details, reproduction recipe, and artifact index
- Creates `.tar.gz` tarball of the entire artifact dir
- Prints artifact path to stdout for CI discovery

**`test_agent_rulez_opencode_capture_to_export`** — `@pytest.mark.slow` test following the same 5-phase structure as the Claude test:
1. Setup via `run_setup(runtime="opencode")` + `assert_phase1_setup()`
2. Wizard session via `run_opencode()` with describe mode prompt
3. Session captured: `assert_phase3_session_captured()` + `assert_phase3_rulez_fired()`
4. Export via `run_opencode()` with export prompt
5. Docs assertion via `assert_phase5_docs()`

Uses `try/finally` pattern — failure artifacts collected in `finally` block, never masking the real error.
