# Plan 03-01: Permission Agents + Commands + Multi-Platform

## Objective

Ship the policy island agent, all three commands, the cross-platform tool name mapping reference, and platform detection in setup.sh — so the wizard runs with zero approval prompts during sessions and installs correctly on Claude Code, OpenCode, Gemini CLI, and Codex. This plan completes the permission and delivery layer; the wizard's core skill logic (Plan 1) and session capture (Plan 2) are prerequisites.

## Context

@plugin/SKILL.md — driving wizard logic; agent delegates all behavior here
@plugin/setup/setup.sh — from Plan 2; extended here with platform detection
@docs/superpowers/specs/2026-03-19-codebase-wizard-design.md — Sections 4–5

## Tasks

### Task 1: Write the Policy Island Agent
Create `plugin/agents/codebase-wizard-agent.md` with scoped `allowed_tools`:
Read/Glob/Grep unrestricted; Write and Edit scoped to `.code-wizard/**` and
`.claude/code-wizard/**` only; Bash limited to node*, grep*, find*, cat*, ls*,
rulez*; no destructive ops; WebSearch and WebFetch pre-authorized. Include body
explaining each permission group and the security rationale.

### Task 2: Write the Main Wizard Command
Create `plugin/commands/codebase-wizard.md` with `context: fork` and
`agent: codebase-wizard-agent` in frontmatter. Document all three args
(--describe / --explore / --file). Describe mode detection routing and which
reference file loads for each mode. Reference SKILL.md as the driving logic.

### Task 3: Write codex-tools.md
Create `plugin/references/codex-tools.md` with the full 4-platform tool name
matrix (Claude Code / OpenCode / Gemini CLI / Codex) for all 8 tool pairs.
Explain why the mapping exists, when to load the file, and include a dedicated
section on Codex's lack of hook support with the manual export instruction.

### Task 4: Add Platform Detection to setup.sh
Extend `plugin/setup/setup.sh` with `RUNTIME="${1:-claude}"` and a case
statement handling all 5 branches (claude / opencode / gemini / codex / all).
The codex branch prints a manual export message and exits with code 77 (the
conventional skip code for unsupported hook setup). Wrap existing Plan 2 logic
in `install_claude()` and extract shared functions.

### Task 5: End-to-End Verification
Verify all four deliverables without making file changes: agent has 15
allowed_tools entries; main command has context:fork and agent reference; codex
tool matrix has all 8 pairs across all 4 platforms; setup.sh case statement has
all 5 branches with exit 77 in the codex branch. Confirm Plan 2 export and setup
commands are still present and unmodified.
