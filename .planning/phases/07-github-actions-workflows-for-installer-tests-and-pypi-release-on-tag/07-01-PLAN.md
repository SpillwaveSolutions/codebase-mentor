Create `.github/workflows/test-installer.yml` with the following exact structure.

The workflow name is "Test Installer". It triggers on `push` to all branches (no branch filter) and on `pull_request`. No `workflow_dispatch` trigger is needed.

Use a build matrix:
```yaml
strategy:
  matrix:
    os: [ubuntu-latest, macos-latest]
    python-version: ["3.11"]
  fail-fast: false
```

`fail-fast: false` ensures both OS results are visible even when one fails — useful for diagnosing macOS-specific path issues.

Runner: `runs-on: ${{ matrix.os }}`

Steps in order:

1. `actions/checkout@v4` — check out the repo
2. `actions/setup-python@v5` with `python-version: ${{ matrix.python-version }}` and `cache: pip`
3. Install the package from source:
   ```yaml
   - name: Install package from source
     run: pip install -e .
   ```
4. Verify the CLI entry point is available:
   ```yaml
   - name: Verify CLI is installed
     run: ai-codebase-mentor --version
   ```
5. Run install and verify plugin files exist:
   ```yaml
   - name: Install codebase-wizard for Claude
     run: ai-codebase-mentor install --for claude
   - name: Verify plugin files exist
     run: |
       test -d "$HOME/.claude/plugins/codebase-wizard" || (echo "Plugin directory missing" && exit 1)
       test -f "$HOME/.claude/plugins/codebase-wizard/plugin.json" || (echo "plugin.json missing" && exit 1)
   ```
   The `test -f` check for `plugin.json` uses the manifest path from the design spec Section 1: `.claude-plugin/plugin.json` inside the plugin directory. Adjust the path to match the actual file layout Phase 6 produces — if the plugin is copied flat, the check is `$HOME/.claude/plugins/codebase-wizard/plugin.json`; if the `.claude-plugin/` subdirectory is preserved, check `$HOME/.claude/plugins/codebase-wizard/.claude-plugin/plugin.json`. Use the flat path as the default unless Phase 6 specifies otherwise. The key invariant is: the directory exists and is non-empty.
6. Run status:
   ```yaml
   - name: Check status (installed)
     run: ai-codebase-mentor status
   ```
7. Run uninstall and verify files are gone:
   ```yaml
   - name: Uninstall codebase-wizard for Claude
     run: ai-codebase-mentor uninstall --for claude
   - name: Verify plugin files removed
     run: |
       test ! -d "$HOME/.claude/plugins/codebase-wizard" || (echo "Plugin directory still exists after uninstall" && exit 1)
   ```
8. Run status again (should report not installed, exit 0 — status is always safe per converter contract):
   ```yaml
   - name: Check status (uninstalled)
     run: ai-codebase-mentor status
   ```

Do NOT include any `secrets`, `env` blocks referencing PYPI tokens, or any publish/release steps. This workflow is test-only.

Do NOT create `.github/workflows/publish-pypi.yml` — that file is explicitly deferred to Milestone 2 (v1.2). Creating it now would violate the phase scope.

The complete workflow YAML:

```yaml
name: Test Installer

on:
  push:
  pull_request:

jobs:
  test-installer:
    name: Test on ${{ matrix.os }} / Python ${{ matrix.python-version }}
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        os: [ubuntu-latest, macos-latest]
        python-version: ["3.11"]
      fail-fast: false

    steps:
      - name: Check out repository
        uses: actions/checkout@v4

      - name: Set up Python ${{ matrix.python-version }}
        uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python-version }}
          cache: pip

      - name: Install package from source
        run: pip install -e .

      - name: Verify CLI entry point is available
        run: ai-codebase-mentor --version

      - name: Install codebase-wizard for Claude
        run: ai-codebase-mentor install --for claude

      - name: Verify plugin files exist
        run: |
          test -d "$HOME/.claude/plugins/codebase-wizard" \
            || (echo "ERROR: plugin directory not found at $HOME/.claude/plugins/codebase-wizard" && exit 1)

      - name: Check status (installed)
        run: ai-codebase-mentor status

      - name: Uninstall codebase-wizard for Claude
        run: ai-codebase-mentor uninstall --for claude

      - name: Verify plugin files removed
        run: |
          test ! -d "$HOME/.claude/plugins/codebase-wizard" \
            || (echo "ERROR: plugin directory still present after uninstall" && exit 1)

      - name: Check status (uninstalled)
        run: ai-codebase-mentor status
```

Write this file to `.github/workflows/test-installer.yml`. Create the `.github/workflows/` directory if it does not exist.
