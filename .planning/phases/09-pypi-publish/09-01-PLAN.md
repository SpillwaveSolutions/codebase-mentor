<task type="auto">
  <name>Task 2: Create publish-pypi.yml workflow</name>
  <files>.github/workflows/publish-pypi.yml</files>
  <read_first>.github/workflows/test-installer.yml</read_first>
  <action>
Create `.github/workflows/publish-pypi.yml` with this exact content:

```yaml
name: Publish to PyPI

on:
  push:
    tags:
      - 'v[0-9]+.[0-9]+.[0-9]+'

permissions:
  id-token: write
  contents: read

jobs:
  publish:
    runs-on: ubuntu-latest
    environment: pypi

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      - name: Build package
        run: |
          pip install build
          python -m build

      - name: Publish to PyPI
        uses: pypa/gh-action-pypi-publish@release/v1
```

This workflow:
- Fires ONLY on semver tag push (v1.2.0, v2.0.0, etc.) -- NOT on branch push or PR
- Uses Trusted Publishers OIDC via `id-token: write` -- no PYPI_TOKEN secret
- Builds sdist + wheel via `python -m build`
- Publishes via `pypa/gh-action-pypi-publish@release/v1` (no `password:` needed)
- Contains ZERO test steps (testing is test-installer.yml's job)
- Uses `environment: pypi` (must match PyPI Trusted Publisher config)
  </action>
