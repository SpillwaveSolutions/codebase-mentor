# Plan 02-01: Capture + Synthesis

## Objective

Build the capture + synthesis pipeline for the Codebase Wizard: an Agent Rulez hook config template that auto-appends Q&A turns to raw session JSON, a setup script that resolves storage, installs the hook runner, and writes scoped permissions, and two commands — `/codebase-wizard-export` to synthesize raw sessions into SESSION-TRANSCRIPT.md, CODEBASE.md, TOUR.md, or FILE-NOTES.md, and `/codebase-wizard-setup` to run the one-time onboarding flow.

## Context

@plugin/SKILL.md — existing wizard skill (Phase 2 builds on top of Phase 1 output)
@docs/superpowers/specs/2026-03-19-codebase-wizard-design.md — Section 3 (hook config, raw session format, synthesis) and Section 5 (commands, settings.local.json)
@plugin/references/describe-questions.md — output schemas for CODEBASE.md
@plugin/references/explore-questions.md — output schemas for TOUR.md

## Tasks

### Task 1: Create plugin/setup/agent-rulez-sample.yaml
- PostToolUse hook: appends turn JSON to `{resolved_storage}/sessions/{date}-{repo}.json`
- `{date}` and `{repo}` documented as Agent Rulez native interpolation variables
- `on_error: warn` so capture failures do not abort sessions
- Stop hook: `notify` action with message to run `/codebase-wizard-export`
- `{resolved_storage}` placeholder substituted by setup.sh at deploy time

### Task 2: Create plugin/setup/setup.sh
- Step 1: Resolve storage — check `.code-wizard/` then `.claude/code-wizard/`; ask user if neither exists; create chosen directory with `sessions/` and `docs/` subdirs
- Step 2: Write `config.json` with `version`, `resolved_storage`, `created`
- Step 3: Install Agent Rulez via `rulez install`
- Step 4: Copy `agent-rulez-sample.yaml` → `{resolved_storage}/agent-rulez.yaml` with `sed` substitution of `{resolved_storage}`
- Step 5: Register hook via `rulez hook add --config {resolved_storage}/agent-rulez.yaml`
- Step 6: Write `settings.local.json` with scoped `Write(.code-wizard/**)` and `Write(.claude/code-wizard/**)` permissions

### Task 3: Create plugin/commands/codebase-wizard-export.md
- `context: fork`, `agent: codebase-wizard-agent` in frontmatter
- Read `config.json` to find `resolved_storage` (check both locations)
- Args: `--latest` (default), `--session <filename>`, `--all`
- Synthesize SESSION-TRANSCRIPT.md for every mode
- Synthesize CODEBASE.md for describe mode, TOUR.md for explore mode, FILE-NOTES.md for file mode
- Output to `{resolved_storage}/docs/{session_id}/`

### Task 4: Create plugin/commands/codebase-wizard-setup.md
- No `context: fork` — runs in main context for broader install permissions
- Comment in frontmatter explains the intentional absence of `context: fork`
- Runs `plugin/setup/setup.sh`
- Documents all 6 steps (resolve → config.json → install → deploy → register → settings.local.json)
- Clearly marked as one-time onboarding; not for regular sessions

## Success Criteria

1. Running `/codebase-wizard-setup` creates storage dir, installs Agent Rulez, and writes `settings.local.json` without any additional approval prompts
2. Every wizard session automatically appends turn data to a dated JSON file in `{resolved_storage}/sessions/` via the PostToolUse hook
3. Running `/codebase-wizard-export --latest` produces SESSION-TRANSCRIPT.md plus the mode-appropriate output doc (CODEBASE, TOUR, or FILE-NOTES) in the correct output directory
4. Running `/codebase-wizard-export --all` processes each session independently without merging across session files
5. Hook capture failures do not abort wizard sessions — `on_error: warn` keeps sessions running and the in-memory buffer provides a fallback
