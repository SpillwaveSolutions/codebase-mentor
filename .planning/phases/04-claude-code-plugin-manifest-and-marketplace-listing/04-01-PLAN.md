```json

  "version": "1.0.0",

Two modes: Describe (repo owner documents their codebase) and Explore (new developer gets a learning-order tour). Sessions auto-captured and synthesizable as CODEBASE.md or TOUR.md.",
  "short_description": "Explore and document any codebase through conversational Q&A with auto-captured sessions.",
  "keywords": ["codebase", "documentation", "exploration", "wizard", "explain", "tour", "ai-codebase-mentor"],

    "name": "Spillwave Solutions",
    "url": "https://github.com/SpillwaveSolutions"

    "type": "git",
    "url": "https://github.com/SpillwaveSolutions/codebase-mentor"
  },
  "runtime": "claude-code",
  "entry": ".claude-plugin/plugin.json",
  "install": {
    "method": "python-package",
    "package": "ai-codebase-mentor",

    {

      "description": "One-time setup: install Agent Rulez hooks and write session-agent permissions."
    },
    {
      "name": "codebase-wizard-export",
      "description": "Synthesize captured session logs into CODEBASE.md, TOUR.md, or FILE-NOTES.md."
    }
  ]
}
```

Notes on field decisions:
- `display_name`: human-readable title with correct casing — "Codebase Wizard"
- `short_description`: one sentence, under 100 chars — used in search result cards
- `tags`: broader discovery categories beyond exact keywords (e.g., "productivity", "developer-tools")
- `entry`: relative path from the marketplace.json location to the install manifest — installer reads this to locate plugin.json
- `install.method`: "python-package" signals the preferred install path; installers can also do a direct copy
- `commands`: inline summaries for the marketplace UI — not the full command files, just name + one-line description
- Do NOT add `rating`, `downloads`, `created_at`, or other server-managed fields — those are populated by the marketplace server, not the plugin author
